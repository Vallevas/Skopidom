# CSV Import для Skopidom

## Описание

Инструмент для массового импорта предметов инвентаря из CSV файлов.

## Формат CSV файла

CSV файл должен содержать следующие колонки (в указанном порядке):

```csv
barcode,inventory_number,name,category,building,room,description
12345,INV-001,Стол письменный,Мебель,Главный корпус,Аудитория 101,Деревянный стол
67890,INV-002,Стул офисный,Мебель,Главный корпус,Аудитория 101,
```

### Обязательные поля:
- `barcode` - серийный номер (уникальный)
- `inventory_number` - инвентарный номер (уникальный)
- `name` - название предмета
- `category` - название категории
- `building` - название здания
- `room` - название комнаты

### Опциональные поля:
- `description` - описание предмета (может быть пустым)

### Требования:
- Кодировка: UTF-8
- Разделитель: запятая (`,`)
- Первая строка - заголовок (обязательно)

## Использование

### 1. Проверка CSV файла (dry-run)

Проверяет корректность CSV файла без импорта данных:

```bash
make seed-dry-run FILE=items.csv
```

### 2. Импорт данных

Импортирует предметы из CSV файла:

```bash
make seed FILE=items.csv USER=admin@university.ru
```

Параметры:
- `FILE` - путь к CSV файлу (относительно директории `backend/`)
- `USER` - email пользователя, от имени которого выполняется импорт

### 3. Прямой запуск команды

Можно также запустить команду напрямую:

```bash
cd backend
go run ./cmd/seed --file=items.csv --user=admin@university.ru
```

Флаги:
- `--file` - путь к CSV файлу (обязательный)
- `--user` - email пользователя (обязательный)
- `--dry-run` - режим проверки без импорта

## Логика работы

### Автоматическое создание справочников

Если категория, здание или комната не существуют в базе данных, они будут созданы автоматически:

- **Категория**: создается с указанным именем
- **Здание**: создается с указанным именем и пустым адресом
- **Комната**: создается с указанным именем в указанном здании

### Обработка дубликатов

Если предмет с таким `barcode` или `inventory_number` уже существует, строка будет пропущена с соответствующим сообщением.

### Audit Log

Все импортированные предметы логируются в `audit_events` как обычное создание предмета (action: `created`).

## Обработка ошибок

### Валидация перед импортом

Все строки проверяются на наличие обязательных полей перед началом импорта. Если найдены ошибки валидации, импорт не начнется.

### Обработка ошибок во время импорта

Каждая строка обрабатывается независимо. Ошибка в одной строке не останавливает импорт остальных.

### Exit код

- `0` - успешный импорт (все строки обработаны без ошибок)
- `1` - импорт завершен с ошибками

## Примеры использования

### Импорт из файла в корне backend/

```bash
make seed FILE=items.csv USER=admin@university.ru
```

### Импорт из подпапки

```bash
make seed FILE=data/items.csv USER=admin@university.ru
```

### Проверка большого файла перед импортом

```bash
make seed-dry-run FILE=large_import.csv
```

## Ограничения

- Максимальный размер CSV файла ограничен только доступной памятью
- Каждая строка обрабатывается в отдельной транзакции
- Импорт выполняется последовательно (не параллельно)

## Устранение неполадок

### "Failed to find user with email"

Убедитесь, что пользователь с указанным email существует в базе данных. Создайте администратора:

```bash
make createadmin
```

### "barcode already exists"

Предмет с таким серийным номером уже существует. Это нормально - строка будет пропущена.

### "Failed to parse CSV"

Проверьте:
- Файл существует по указанному пути
- Файл в кодировке UTF-8
- Первая строка содержит корректный заголовок
- Все строки содержат 7 колонок
